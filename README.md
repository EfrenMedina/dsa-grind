# dsa-grind

My daily grind through data structures & algorithms — **2 problems a day**, currently working the [NeetCode 150](https://neetcode.io/practice).

Solutions are in Python. Each problem lives in its own folder so it can hold both the solution and (later) a short writeup of the idea and common pitfalls.

## Structure

```
<category>/<leetcode#>-<slug>/
    solution.py     # the solution
    notes.md        # (optional, added later) intuition + pitfalls
```

Categories follow the NeetCode roadmap: arrays-hashing, two-pointers, sliding-window,
stack, binary-search, linked-list, trees, tries, backtracking, heap-priority-queue,
graphs, advanced-graphs, 1d-dp, 2d-dp, greedy, intervals, math-geometry, bit-manipulation.

See [`_template/`](_template) for the starting point I copy for each new problem.
